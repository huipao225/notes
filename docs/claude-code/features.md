# Claude Code 功能详细介绍

## 概述

Claude Code 是 Anthropic 推出的命令行 AI 编程助手，直接在终端中运行，能够理解你的代码库、回答技术问题、编写和修改代码、执行命令等。以下从实际使用场景出发，逐一介绍其核心能力。

---

## 1. 代码阅读与理解

### 读取文件
可以读取任意文本文件、代码文件，甚至图片和 PDF。

```
用户：帮我看看 src/main.c 里做了什么
Claude：读取文件，逐段分析 main 函数的逻辑
```

### 理解项目结构
自动浏览目录树、理解模块依赖关系、分析头文件包含链。

```
用户：这个项目里哪些文件依赖了 network.h？
Claude：全局搜索 #include "network.h"，列出所有引用文件
```

### 阅读图片
支持 PNG、JPG 等常见图片格式，可以读取架构图、流程图、截图中的报错信息。

```
用户：帮我看看这张报错截图是什么问题（附图片路径）
Claude：读取图片内容，识别报错文本并分析原因
```

---

## 2. 代码搜索与探索

### 文件名搜索（Glob）
按通配符模式匹配文件名。

```bash
# 查找所有 .c 源文件
**/*.c

# 查找 src 目录下所有头文件
src/**/*.h

# 查找名字包含 test 的文件
**/*test*.*
```

### 内容搜索（Grep）
支持正则表达式的全文搜索，比 grep 更快更准。

```
用户：找出所有调用 malloc 但没检查返回值的地方
Claude：用正则搜索 malloc( 附近的代码，检查是否有 NULL 判断
```

### 符号定位
快速定位函数定义、变量声明、宏定义等。

```
用户：swap 函数是在哪个文件里定义的？
Claude：搜索 void swap( 定位到具体文件和行号
```

---

## 3. 代码编辑与写入

### 精确编辑（Edit）
对现有文件做精确的字符串替换，不会重写整个文件。

```
用户：把 main.c 里所有 int 类型的循环变量改成 size_t
Claude：找到 for (int i = 0; ...) 替换为 for (size_t i = 0; ...)
```

### 创建新文件（Write）
从零创建代码文件、配置文件、文档等。

```
用户：帮我创建一个 Makefile，编译 src 下所有 .c 文件
Claude：分析项目结构后生成对应的 Makefile
```

### 批量修改
使用 `replace_all` 一次性修改文件中所有匹配项。

```
用户：把这个文件里的 snake_case 全改成 camelCase
Claude：逐个变量名进行全局替换
```

---

## 4. 终端命令执行

### 编译与构建
直接执行 gcc、make、cmake、npm 等构建命令。

```
用户：帮我把当前项目编译一下
Claude：执行 gcc -std=c99 -Wall -o main main.c，报告编译结果
```

### 运行与测试
执行程序、运行测试套件、查看运行输出。

```
用户：跑一下测试，看看哪些用例没通过
Claude：执行测试命令，整理失败用例清单并分析原因
```

### 安装依赖
安装所需的库、工具链等。

```
用户：帮我装一下项目需要的依赖
Claude：根据 package.json / CMakeLists.txt 判断依赖并执行安装命令
```

### Git 操作
查看状态、查看差异、创建提交、创建分支等。

```
用户：看看我改了哪些文件，帮我写个合适的 commit message
Claude：执行 git diff 和 git status，分析修改内容后生成规范的提交信息
```

### 后台执行
耗时长的命令可以放到后台运行，完成后自动通知。

```
用户：在后台编译这个大型项目，好了告诉我
Claude：后台执行 make -j8，编译完成后汇报结果
```

---

## 5. 任务规划与追踪

### 复杂任务拆解
当任务涉及多步骤、多文件时，Claude 会先规划再执行。

```
用户：给我这个项目加上日志功能
Claude：
  1. 先了解项目结构
  2. 制定方案（日志用什么格式、写到哪里）
  3. 创建计划等待你确认
  4. 逐一实现各个步骤
```

### 任务进度追踪
自动创建任务列表，完成后逐个打勾，让用户随时了解进度。

---

## 6. 多智能体协作（Agent）

Claude Code 可以派生子智能体并行处理多个独立任务。

### 探索智能体（Explore）
专门用于大范围代码搜索和探索，比手动搜索更高效。

```
用户：帮我找出项目里所有没有错误处理的网络调用
Claude：派探索智能体扫描所有相关文件，汇总结果
```

### 审查智能体
独立审查代码变更，提供第二意见。

```
用户：帮我审查一下这次改动有没有安全问题
Claude：派审查智能体检查所有修改，报告潜在风险
```

### 计划智能体（Plan）
负责设计实现方案，输出分步骤的实施计划。

```
用户：我想把单线程程序改成多线程，帮我做个方案
Claude：派计划智能体分析现有代码，输出改造计划
```

### 并行处理
多个独立任务可以同时派多个智能体并行处理。

```
用户：同时检查 backend/ 和 frontend/ 两个目录的代码质量问题
Claude：开启两个智能体，一个查后端一个查前端，同时进行
```

---

## 7. 计划模式（Plan Mode）

进入计划模式后，Claude 会：
- 深入探索代码库，理解现有架构
- 设计多种实施方案，分析优劣势
- 识别需要修改的关键文件
- 输出详细的实施计划
- 等待你确认后再动手写代码

```
用户：我想把这个程序改成客户端-服务器架构
Claude：进入计划模式 → 分析现有代码 → 设计架构方案 → 等你确认 → 开始实现
```

---

## 8. 记忆系统

Claude Code 拥有持久化记忆，可以在多次会话之间记住重要信息。

### 记忆类型
- **用户信息**：你的角色、偏好、技术背景
- **反馈记录**：你对工作方式的偏好（"不要这样做"、"这样做很好"）
- **项目信息**：当前项目的目标、背景、注意事项
- **参考资料**：外部系统链接（文档地址、API 文档等）

```
用户：记住，这个项目用的是 C99 标准，别用 C11 的特性
Claude：保存到记忆，以后写代码自动遵循 C99
```

```
用户：上次你说的那个 xxx 方案，这次还用吗？
Claude：查阅记忆，回顾上次的方案决策
```


## 10. 斜杠命令（Slash Commands）

在 Claude Code 终端中直接输入的命令：

| 命令 | 功能 |
|------|------|
| `/help` | 查看帮助 |
| `/init` | 为当前项目生成 CLAUDE.md 配置文件 |
| `/clear` | 清空当前会话上下文 |
| `/compact` | 压缩上下文（当对话过长时释放空间） |
| `/context` | 查看当前上下文使用情况 |
| `/rewind` | 回滚到之前的状态 |
| `/effort` | 切换思考强度（平衡速度与深度） |
| `/exit` | 退出 Claude Code |
| `/resume` | 查看最近的会话历史 |
| `/review` | 审查当前分支的代码变更 |
| `/security-review` | 对当前修改做安全检查 |

### 终端切换
在 Claude 对话中，输入 `!` 可以临时切换到终端执行命令；`shift+tab` 切换权限模式。

## 11. Git 集成

### 提交代码
自动分析修改内容，生成规范的提交信息。

```
用户：帮我提交这些改动
Claude：
  1. 运行 git status 查看改动文件
  2. 运行 git diff 分析具体修改
  3. 生成符合项目风格的 commit message
  4. 执行 git add 和 git commit
```

### 创建 Pull Request
从分析改动到推送分支到创建 PR，全流程自动化。

```
用户：把当前分支推到 GitHub 并发起 PR
Claude：
  1. 对比 base 分支，汇总所有改动
  2. 生成 PR 标题和描述
  3. 推送分支
  4. 用 gh pr create 创建 PR
```

## 12. 定时任务（Cron）

可以设置定期执行的任务。

```
用户：每 5 分钟检查一次编译是否成功
Claude：设置定时任务，每 5 分钟执行一次 make，失败时通知你
```

```
用户：每天早上 9 点提醒我 review 昨天的提交
Claude：设置工作日 8:57 的定时提醒
```

---

## 13. 工作树隔离（Worktree）

创建隔离的 Git 工作树，在不影响当前分支的情况下进行实验性修改。

```
用户：开个工作树，我想试试用另一种算法重写这个函数
Claude：创建隔离工作树 → 在新环境中修改 → 不影响原分支
```

---

## 14. 技能系统（Skills）

可加载的专业技能模块，针对特定场景提供深度支持：

- **claude-api**：构建和调试 Claude API / Anthropic SDK 应用，处理模型迁移、prompt 缓存等
- **security-review**：对当前分支做完整的安全审查
- **review**：审查 PR 代码变更
- **simplify**：检查代码的复用性、质量和效率

## 15. 实际工作流示例

### 示例一：修复 Bug
```
用户：程序跑起来就崩溃，帮我看看
Claude：
  1. 阅读相关源代码
  2. 搜索可疑的内存操作（未初始化指针、越界访问等）
  3. 必要用 gdb 运行，观察崩溃位置
  4. 定位根因，修改代码
  5. 重新编译验证
```

### 示例二：添加新功能
```
用户：给这个计算器程序加上求幂运算
Claude：
  1. 阅读 main.c 了解现有架构（如何解析输入、如何分发运算）
  2. 确定修改点：解析层加 ^ 运算符、运算层加 power 函数
  3. 逐个文件编辑
  4. 编译测试 2^3、2.5^2 等用例
```

### 示例三：代码重构
```
用户：把项目里散落的错误处理统一一下
Claude：
  1. 全局搜索现有的错误处理模式
  2. 设计统一的 error_handler 模块
  3. 逐一替换各文件中的自定义错误处理
  4. 编译验证，确保行为不变
```

### 示例四：学习与解释
```
用户：解释一下这个链表反转函数是怎么工作的
Claude：逐步分析指针变化过程，画出示意图（文字描述），说明每次循环后链表的状态
```

---

## 总结

Claude Code 的核心能力一句话概括：**它能像人类程序员一样阅读、理解、搜索、修改代码，同时还能执行命令、管理 Git、搜索网络、并行处理多个任务**。它不是简单的代码补全工具，而是一个可以独立完成复杂软件工程任务的 AI 助手。
